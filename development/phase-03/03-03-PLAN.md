**Step 3: Add /admin-link and /admin-unlink handlers to the slash command workflow**

Modify the "03-02 Aerys Discord Slash Commands" workflow (GET → patch → PUT) to add two new routes in RouteCommand Switch:

**Both admin commands share the same first two nodes:**

**Node: Check Admin Role** (HTTP Request node)
```
Method: GET
URL: https://discord.com/api/v10/guilds/{{ $json.guild_id }}/members/{{ $json.userId }}
Credential: YOUR_DISCORD_BOT_CREDENTIAL_ID (Discord Bot API)
```
Do NOT rely on the trigger payload's roles — always fetch via REST (RESEARCH.md Pitfall 4: katerlol doesn't guarantee guild member data in event payload).

**Node: Evaluate Role** (Code node)
```javascript
const member = $input.item.json;
const adminRoleId = $vars.AERYS_ADMIN_ROLE_ID;
const roles = member.roles || [];
const isAdmin = roles.includes(adminRoleId);

// Carry forward the original interaction data
const interaction = $('ParseInteraction').item.json;
return [{ json: { ...interaction, _guild_member: member, _is_admin: isAdmin } }];
```

**Node: Admin Gate** (IF node)
- True (`_is_admin === true`) → admin sub-command handler
- False → **RespondUnauthorized** (Respond to Webhook): `{"type":4,"data":{"content":"You don't have permission to use admin commands.","flags":64}}`

---

**/admin-link handler (after gate passes):**

1. **Lookup Discord Identity** (Postgres Execute Query):
```sql
SELECT pi.person_id::text, p.display_name
FROM platform_identities pi
JOIN persons p ON p.id = pi.person_id
WHERE pi.platform = 'discord' AND pi.platform_user_id = $1
LIMIT 1;
```
Query Parameters: `={{ [$json.options.discord_id] }}`

2. **Lookup Telegram Identity** (Postgres Execute Query):
```sql
SELECT pi.person_id::text, p.display_name
FROM platform_identities pi
JOIN persons p ON p.id = pi.person_id
WHERE pi.platform = 'telegram' AND pi.platform_user_id = $1
LIMIT 1;
```
Query Parameters: `={{ [$json.options.telegram_id] }}`

3. **Prepare Merge** (Code node): Discord person is winner, Telegram is loser. If either doesn't exist yet, handle gracefully (create the missing platform_identities row rather than merging).

4. **Execute Merge** (6 sequential Postgres Execute Query nodes — same pattern as /link redeem):
   - UPDATE platform_identities SET person_id = winner WHERE person_id = loser
   - UPDATE conversations SET person_id = winner WHERE person_id = loser
   - UPDATE messages SET person_id = winner WHERE person_id = loser
   - UPDATE memories SET person_id = winner WHERE person_id = loser
   - UPDATE persons SET deleted_at = NOW() WHERE id = loser
   - DELETE FROM pending_links WHERE person_id = loser (cleanup)

5. **Notify Discord user** (HTTP Request POST):
```
POST https://discord.com/api/v10/users/@me/channels
Body: {"recipient_id": "[discord_id]"}
```
Then POST to the returned DM channel: `{"content":"An admin has linked your Discord account to a Telegram account."}`
(If DM fails — user has DMs disabled — skip silently; reply to admin notes it)

6. **Notify Telegram user** (HTTP Request POST — direct Telegram API, bypassing Output Router):
```
POST https://api.telegram.org/bot{token}/sendMessage
Body: {"chat_id": "[telegram_id]", "text": "An admin has linked your Telegram account to a Discord account."}
```
Fetch the Telegram bot token from the n8n credential store or from an existing Telegram workflow node's hardcoded value. Use `$vars` or a Set node upstream to inject it (process.env blocked in Code nodes).

7. **RespondAdminLink** (Respond to Webhook):
`{"type":4,"data":{"content":"Force-link complete. Discord [discord_id] and Telegram [telegram_id] are now unified.","flags":64}}`

---

**/admin-unlink handler (after gate passes):**

SELECT pi.person_id::text FROM platform_identities

```
Query Parameters: `={{ [$json.options.platform, $json.options.id] }}`

2. **Check found** (IF node)
   - Not found → RespondToWebhook: "No platform identity found for [platform] ID [id]."

3. **Delete row** (Postgres Execute Query):

DELETE FROM platform_identities WHERE platform = $1 AND platform_user_id = $2;
```

4. **RespondAdminUnlink** (Respond to Webhook):
`{"type":4,"data":{"content":"Unlinked [platform] account [id]. They will get a new identity on their next message.","flags":64}}`
