  <name>Task 1: Set up Cloudflare Tunnel on Tachyon and configure Discord Interactions Endpoint</name>
  <files>~/aerys/config/cloudflare/config.yml</files>
  <action>
Aerys needs a public HTTPS endpoint so Discord can POST slash command interactions to n8n. This task installs cloudflared on the Tachyon, creates a named tunnel, routes your-domain.example.com to localhost:5678, and registers the Interactions Endpoint URL with Discord.

**Step 1: Install cloudflared on the Tachyon**

The Tachyon is ARM-based. Detect architecture and install accordingly:

```bash
# Check architecture
ssh localhost "uname -m"
# arm64 / aarch64 → use arm64 binary
# armv7l → use arm binary

# Install ARM64
ssh localhost "curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64 -o /tmp/cloudflared && sudo install /tmp/cloudflared /usr/local/bin/cloudflared && cloudflared --version"

# OR ARM32
ssh localhost "curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm -o /tmp/cloudflared && sudo install /tmp/cloudflared /usr/local/bin/cloudflared && cloudflared --version"
```

**Step 2: Login to Cloudflare (MANUAL BROWSER STEP)**

Run on Tachyon — this prints a URL for the user to open in a browser:
```bash
ssh localhost "cloudflared tunnel login"
```
User opens the printed URL, selects the your-domain.work zone, authorizes. The cert is saved to `~/.cloudflared/cert.pem` on Tachyon. Confirm with:
```bash
ssh localhost "ls ~/.cloudflared/cert.pem"
```

**Step 3: Create the tunnel**

```bash
ssh localhost "cloudflared tunnel create aerys"
```

Note the tunnel UUID from output (e.g. `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`). Save it:
```bash
TUNNEL_UUID=$(ssh localhost "cloudflared tunnel list --output json" | python3 -c "import sys,json; t=[t for t in json.load(sys.stdin) if t['name']=='aerys']; print(t[0]['id'])")
echo "Tunnel UUID: $TUNNEL_UUID"
```

**Step 4: Write config file**

```bash
ssh localhost "sudo mkdir -p /etc/cloudflared"
ssh localhost "sudo tee /etc/cloudflared/config.yml > /dev/null << EOF
tunnel: ${TUNNEL_UUID}
credentials-file: /root/.cloudflared/${TUNNEL_UUID}.json

ingress:
  - hostname: your-domain.example.com
    service: http://localhost:5678
  - service: http_status:404
EOF"
```

Also save a copy to the infra repo:
```bash
mkdir -p ~/aerys/config/cloudflare
ssh localhost "cat /etc/cloudflared/config.yml" > ~/aerys/config/cloudflare/config.yml
```

Note: credentials file lives at `/root/.cloudflared/${TUNNEL_UUID}.json` — cloudflared login stores it there when run as root, or `~/.cloudflared/` when run as the current user. Check actual path:
```bash
ssh localhost "ls ~/.cloudflared/*.json 2>/dev/null || ls /root/.cloudflared/*.json 2>/dev/null"
```
Update the config.yml credentials-file path to match.

**Step 5: Create DNS record**

```bash
ssh localhost "cloudflared tunnel route dns aerys your-domain.example.com"
```

**Step 6: Install and start as systemd service**

```bash
ssh localhost "sudo cloudflared service install && sudo systemctl enable cloudflared && sudo systemctl start cloudflared"
ssh localhost "sudo systemctl status cloudflared"
```

**Step 7: Verify tunnel is healthy**

```bash
# Check tunnel status
ssh localhost "cloudflared tunnel info aerys"

# Test public endpoint reaches n8n
curl -s -o /dev/null -w "%{http_code}" "https://your-domain.example.com/healthz"
# Expected: 200
```

**Step 8: Configure Discord Interactions Endpoint URL (MANUAL BROWSER STEP)**

User opens: https://discord.com/developers/applications/YOUR_DISCORD_APPLICATION_ID/information

Under "Interactions Endpoint URL", enter:
```
https://your-domain.example.com/webhook/discord/aerys-commands
```

Discord will POST a PING (type 1) to validate it. The Aerys Discord Slash Commands workflow (built in Plan 02) must be active and running for this validation to succeed. **Defer this step until after Plan 02 Task 2 is complete** — come back here then.

**Step 9: Commit config to infra branch**

```bash
cd ~/aerys && git add config/cloudflare/config.yml && git commit -m "feat(03): add cloudflare tunnel config for your-domain.example.com" && git push
```

<task type="auto">
  <name>Task 2: Run DB migration 002_identity.sql</name>
  <files>~/aerys/migrations/002_identity.sql</files>
  <action>
Write ~/aerys/migrations/002_identity.sql with the following DDL, then apply it to the live database via `docker exec`:

```sql
\c aerys

-- Extensible platform identity store
-- Authoritative store for all platform->person mappings from Phase 3 forward.
-- The existing persons.discord_id and persons.telegram_id columns are kept for
-- backward compat but platform_identities is the canonical reference.
CREATE TABLE IF NOT EXISTS platform_identities (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    person_id        UUID NOT NULL REFERENCES persons(id) ON DELETE CASCADE,
    platform         TEXT NOT NULL,           -- 'discord', 'telegram'
    platform_user_id TEXT NOT NULL,           -- raw platform ID (string always)
    username         TEXT,                    -- display name from platform at link time
    linked_at        TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (platform, platform_user_id)       -- one account per platform per person
);

CREATE INDEX IF NOT EXISTS idx_platform_identities_person
    ON platform_identities(person_id);

CREATE INDEX IF NOT EXISTS idx_platform_identities_lookup
    ON platform_identities(platform, platform_user_id);

-- Short-lived verification code store for user-initiated cross-platform linking
CREATE TABLE IF NOT EXISTS pending_links (
    id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code       TEXT NOT NULL UNIQUE,
    person_id  UUID NOT NULL REFERENCES persons(id) ON DELETE CASCADE,
    platform   TEXT NOT NULL,                -- platform where code was issued
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_pending_links_code
    ON pending_links(code);

CREATE INDEX IF NOT EXISTS idx_pending_links_expires
    ON pending_links(expires_at);
```

Apply with:
```bash
docker exec -i aerys-postgres-1 psql -U n8n_admin -d aerys < ~/aerys/migrations/002_identity.sql
```

```bash
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "\d platform_identities"
```

  <name>Task 3: Build identity resolver sub-workflow (03-01)</name>
  <files>~/aerys/workflows/03-01-identity-resolver.json</files>
  <action>
Create a new n8n workflow named "03-01 Identity Resolver" via the n8n API. This is a sub-workflow (uses "Execute Sub-workflow Trigger" as its entry point). It is called synchronously by both platform adapters (for chat messages) and by the Discord slash command workflow (for commands).

**Workflow node chain:**

1. **Execute Sub-workflow Trigger** (built-in)
   - Entry point — receives `{platform, platform_user_id, username}` from caller

2. **Cleanup Expired Codes** (Postgres — Execute Query)
   - Credential: YOUR_POSTGRES_CREDENTIAL_ID (Postgres aerys DB)
   - SQL: `DELETE FROM pending_links WHERE expires_at < NOW();`

3. **Lookup Identity** (Postgres — Execute Query)
   - Credential: YOUR_POSTGRES_CREDENTIAL_ID
   - SQL:
     ```sql
     SELECT pi.person_id::text, p.display_name
     FROM platform_identities pi
     JOIN persons p ON p.id = pi.person_id
     WHERE pi.platform = $1 AND pi.platform_user_id = $2
     LIMIT 1;
     ```
   - Query Parameters (as array): `={{ [$json.platform, $json.platform_user_id] }}`

4. **Check Found** (IF node)
   - Condition: `{{ $json.person_id }}` is not empty
   - True branch → **Return Existing** (Set node)
   - False branch → **Create Person** (Postgres — Execute Query)

5. **Return Existing** (Set node — true branch)
   - Fields: `person_id` = `{{ $json.person_id }}`, `display_name` = `{{ $json.display_name }}`, `is_new` = `false`

6. **Create Person** (Postgres — Execute Query — false branch)
   - Credential: YOUR_POSTGRES_CREDENTIAL_ID
   - SQL:
     ```sql
     INSERT INTO persons (display_name)
     VALUES ($1)
     RETURNING id::text, display_name;
     ```
   - Query Parameters: `={{ [$('Execute Sub-workflow Trigger').item.json.username || 'Unknown'] }}`

7. **Create Platform Identity** (Postgres — Execute Query)
   - Credential: YOUR_POSTGRES_CREDENTIAL_ID
   - SQL:
     ```sql
     INSERT INTO platform_identities (person_id, platform, platform_user_id, username)
     VALUES ($1, $2, $3, $4)
     ON CONFLICT (platform, platform_user_id) DO NOTHING
     RETURNING person_id::text;
     ```
   - Query Parameters: `={{ [$json.id, $('Execute Sub-workflow Trigger').item.json.platform, $('Execute Sub-workflow Trigger').item.json.platform_user_id, $('Execute Sub-workflow Trigger').item.json.username || 'Unknown'] }}`

8. **Return New** (Set node — after create)
   - Fields: `person_id` = `{{ $('Create Person').item.json.id }}`, `display_name` = `{{ $('Create Person').item.json.display_name }}`, `is_new` = `true`
