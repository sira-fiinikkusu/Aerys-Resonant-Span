YouTube Innertube (Code node — no API key needed):
```javascript
// Step 1: Get video page to find captions
const videoId = $json.videoId;
const page = await $http.request({
  method: 'GET',
  url: `https://www.youtube.com/watch?v=${videoId}`,
  headers: {'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'}
});
// Extract ytInitialPlayerResponse from page HTML
const match = page.body.match(/ytInitialPlayerResponse\s*=\s*(\{.+?\})\s*;\s*(?:var|const|let)\s/);
if (!match) throw new Error('Could not parse YouTube page');
const playerResponse = JSON.parse(match[1]);
const captions = playerResponse?.captions?.playerCaptionsTracklistRenderer?.captionTracks || [];
if (captions.length === 0) return [{json: {transcript: null, fallback: true, title: playerResponse?.videoDetails?.title}}];
// Step 2: Fetch caption track (baseUrl is the direct URL to timedtext)
const captionUrl = captions[0].baseUrl;
const captionXml = await $http.request({method: 'GET', url: captionUrl});
// Strip XML tags to get plain text
const text = captionXml.body.replace(/<[^>]+>/g, ' ').replace(/&amp;/g,'&').replace(/&#39;/g,"'").replace(/&quot;/g,'"').trim();
return [{json: {transcript: text, videoId, title: playerResponse?.videoDetails?.title}}];
```
YouTube transcript escalation sequence (try in order — do NOT skip to graceful dead-end without attempting all three):
1. **Innertube raw** (above) — try first, no dependencies
2. **youtubei.js npm package** — if Innertube raw fails (API changed, parse error). Requires `NODE_FUNCTION_ALLOW_EXTERNAL=youtubei.js` added to docker-compose.yml n8n environment. Add the env var and rebuild (`docker compose up -d`) before trying. This is a proper maintained Innertube client (https://github.com/LuanRT/YouTube.js).
3. **RapidAPI YouTube transcript** — if youtubei.js also fails. Paid API, HTTP Request from Code node, no npm needed. Get a key from RapidAPI dashboard (youtube-transcript or similar endpoint). Add `RAPIDAPI_KEY` to n8n Variables.
4. **Graceful dead-end** — only if all three fail: return video title + "I couldn't fetch the transcript for this one. If you paste the key points, I'll work with that." This keeps the tool alive but transparent about its limits.
Note in 05-01-SUMMARY which approach succeeded and which failed — this informs any future YouTube-related work.

Create the 05-01 media sub-agent workflow in n8n via the API. Structure:

**Entry + type detection:**
1. Execute Workflow Trigger (typeVersion 1, passthrough)
2. Detect Media Type (Code node):
   - Read `const input = $('Execute Workflow Trigger').first().json`
   - Check attachments array for image (jpg/jpeg/png/webp/gif or content_type startsWith 'image/'), pdf (.pdf), docx (.docx), txt (.txt)
   - Check content for YouTube URL pattern: `/(?:youtube\.com\/watch\?v=|youtu\.be\/)([a-zA-Z0-9_-]{11})/`
   - Set: {mediaType: 'image'|'pdf'|'docx'|'txt'|'youtube'|'unknown', mediaUrl, videoId, filename, platform, content, person_id}
   - For Telegram: attachments come as file_id + mime_type (from katerlol normalized payload) — set needsTelegramDownload: true
3. Route by Media Type (Switch node, 5 branches: image, pdf, docx, txt, youtube + fallback to 'unknown')

**Image branch — model selection with fallback (LOCKED DECISION: best available, not hardcoded):**
4a. Build Vision Request (Code node):
   ```javascript
   const input = $('Execute Workflow Trigger').first().json;
   // Priority-ordered list — use first model that succeeds.
   // LOCKED DECISION: "Best available vision model via OpenRouter (not hardcoded to Gemini Flash)."
   const VISION_MODELS = [
     'google/gemini-2.0-flash-exp:free',  // preferred: best free vision
     'google/gemini-flash-1.5',            // paid fallback
     'anthropic/claude-3-haiku'            // final fallback — broad availability
   ];
   return [{json: {
     vision_models: VISION_MODELS,
     current_model_index: 0,
     prompt: input.content || 'Describe this image in detail.',
     imageUrl: input.mediaUrl,
     platform: input.platform,
     person_id: input.person_id,
     max_tokens: 1024
   }}];
   ```
4b. If platform = 'telegram': Telegram Get File (built-in node, file_id) → HTTP Request GET to download binary → Convert to base64 in Code node → replace imageUrl with data URI (`data:image/jpeg;base64,{b64}`)
4c. Call OpenRouter Vision (HTTP Request POST to https://openrouter.ai/api/v1/chat/completions, credential YOUR_OPENROUTER_CREDENTIAL_ID):
   - Body built from Build Vision Request output:
     ```json
     {
       "model": "={{$json.vision_models[$json.current_model_index]}}",
       "messages": [{"role": "user", "content": [{"type": "text", "text": "={{$json.prompt}}"}, {"type": "image_url", "image_url": {"url": "={{$json.imageUrl}}"}}]}],
       "max_tokens": 1024
     }
     ```
   - onError: continueRegularOutput (so fallback logic can run)
4d. Handle Vision Fallback (Code node — runs after HTTP Request):
   ```javascript
   const response = $input.first().json;
   const request = $('Build Vision Request').item.json;
   // If error or no choices, try next model in list
   if (response.error || !response.choices || response.choices.length === 0) {
     const nextIndex = request.current_model_index + 1;
     if (nextIndex < request.vision_models.length) {
       // Signal retry with next model — executor loops back to step 4c
       // Implementation: use a Loop node or re-execute HTTP Request with nextIndex
       return [{json: {...request, current_model_index: nextIndex, retry: true}}];
     }
     // All models exhausted — return graceful error
     return [{json: {vision_description: null, vision_error: true, model_tried: request.vision_models[request.current_model_index]}}];
   }
   return [{json: {
     vision_description: response.choices[0].message.content,
     model_used: response.model || request.vision_models[request.current_model_index],
     vision_error: false
   }}];
   ```
   Note: If n8n loop construct is unavailable, implement as sequential HTTP Requests for each fallback model gated by IF nodes (try model 1 → if error → try model 2 → if error → try model 3 → if all fail, return graceful error). The important constraint is that no single model is hardcoded as the only attempt.

**PDF branch:**
5a. Download PDF (HTTP Request GET mediaUrl, response format: file/binary)
5b. Extract From File node (operation: pdf) → returns text
5c. Truncate if >8000 chars (Code node — LLM context limit)

**DOCX branch:**
6a. Download DOCX (HTTP Request GET mediaUrl, response format: file/binary)
6b. Use whichever DOCX node was confirmed in 05-00 (community node OR Code node with require('mammoth').extractRawText({buffer: Buffer.from($binary.data, 'base64')}) if NODE_FUNCTION_ALLOW_EXTERNAL=mammoth is set)
6c. Truncate if >8000 chars

**TXT branch:**
7a. Download TXT (HTTP Request GET mediaUrl, response text)
7b. Truncate if >8000 chars

**Merge extracted content:**
8. Merge (n8n Merge node, append mode) — combines output from all document branches
9. Synthesize with LLM (HTTP Request POST to OpenRouter, model: anthropic/claude-haiku-4.5):
   - System: "You are Aerys. Synthesize the following extracted content in response to the user's request. Be concise and natural. If the user asked a question, answer it using the content."
   - User: `User asked: {input.content}\n\nExtracted content from {filename}:\n{extracted_text}`
   - max_tokens: 800

**Return:**
10. Format Return (Code node):
   - return [{json: {result: synthesized_text, media_type: mediaType, filename}}]
