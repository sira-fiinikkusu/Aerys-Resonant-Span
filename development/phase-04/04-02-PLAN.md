3. **Build Retrieval Query** (Code node) — prepare parameters for the retrieval SQL:
```javascript
const input = $input.first().json;
const embeddingResult = $('Embed Query Text').first().json;
const embedding = embeddingResult.data?.[0]?.embedding || [];

// Determine allowed privacy levels based on conversation context
const privacyContext = input.privacy_context || 'public';
const allowedLevels = privacyContext === 'private'
  ? ['public', 'private']   // DM context: can see both
  : ['public'];              // Guild context: only public memories

return [{
  json: {
    person_id: input.person_id,
    embedding_str: '[' + embedding.join(',') + ']',
    allowed_levels: allowedLevels,
    privacy_context: privacyContext
  }
}];
```

4. **Retrieve Memories** (Postgres node, credential YOUR_POSTGRES_CREDENTIAL_ID, operation: executeQuery):
```sql
SELECT
  m.id,
  m.content,
  m.source_platform,
  m.privacy_level,
  m.created_at,
  (1 - (m.embedding <=> $1::vector)) * 0.7
  + LEAST(1.0, GREATEST(0.0,
      1 - EXTRACT(EPOCH FROM (NOW() - m.created_at)) / 2592000.0
    )) * 0.3 AS combined_score
FROM memories m
WHERE m.person_id = $2
  AND m.deleted_at IS NULL
  AND m.privacy_level = ANY($3::text[])
ORDER BY combined_score DESC
LIMIT 5;
```
- Query replacement (options.queryReplacement): `["={{ $json.embedding_str }}", "={{ $json.person_id }}", "={{ JSON.stringify($json.allowed_levels) }}"]`

   **Note on $3 array binding:** If n8n Postgres node has trouble with `ANY($3::text[])` syntax, fallback: build the WHERE clause dynamically in the Code node above and use separate queries for public-only vs public+private.

5. **Format Memory Context** (Code node):
```javascript
const rows = $input.all().map(i => i.json);

if (rows.length === 0) {
  return [{ json: { memory_context: '' } }];
}

// Format as a prose-adjacent bullet list (works well for LLM attention)
const lines = rows.map((r, idx) => {
  const age = r.created_at
    ? `(${Math.round((Date.now() - new Date(r.created_at)) / 86400000)}d ago)`
    : '';
  const src = r.source_platform ? `[${r.source_platform}]` : '';
  return `• ${r.content} ${src} ${age}`.trim();
});

const memoryContext = `${lines.join('\n')}`;
return [{ json: { memory_context: memoryContext } }];
```

**Part C: Create hourly batch extraction workflow (04-02-memory-batch)**

Create a new n8n workflow via POST named "04-02 Memory Batch Extraction".

**Node structure:**

1. **Schedule Trigger** (n8n-nodes-base.scheduleTrigger, interval: 1 hour, mode: everyHour)

2. **Pre-check Message Source** (Postgres node, credential YOUR_POSTGRES_CREDENTIAL_ID, executeQuery — runs FIRST before any other batch logic):
```sql
SELECT
  (SELECT COUNT(*) FROM information_schema.tables WHERE table_name = 'messages') AS messages_table_exists,
  (SELECT COUNT(*) FROM information_schema.columns WHERE table_name = 'messages' AND column_name = 'content') AS messages_has_content,
  (SELECT COUNT(*) FROM information_schema.columns WHERE table_name = 'messages' AND column_name = 'person_id') AS messages_has_person_id;
```

3. **Route to Message Source** (IF node on `$json.messages_has_content > 0 AND $json.messages_has_person_id > 0`):
   - YES path → **Fetch from messages table** (Step 4a below)
   - NO path → **Fetch from n8n_chat_histories** (Step 4b below)

4a. **Fetch Unprocessed Messages — messages table** (Postgres node, credential YOUR_POSTGRES_CREDENTIAL_ID):
```sql
SELECT m.id, m.person_id, m.content, m.source_channel AS source_platform,
       m.conversation_privacy AS privacy_level, m.created_at,
       p.display_name AS speaker_name
FROM messages m
LEFT JOIN persons p ON p.id = m.person_id
WHERE m.processed_at IS NULL
  AND m.created_at >= NOW() - INTERVAL '2 hours'
  AND m.content IS NOT NULL AND m.content != ''
  AND (m.is_bot IS NULL OR m.is_bot = false)
ORDER BY m.created_at ASC
LIMIT 200;
```

4b. **Fetch from n8n_chat_histories — fallback** (Postgres node, credential YOUR_POSTGRES_CREDENTIAL_ID):

**IMPORTANT — attribution note:** `n8n_chat_histories` session_id is a bare `person_id` UUID (Phase 4 design — person_id for all contexts). JOIN directly to `persons` table on `id = session_id::uuid`. Platform cannot be determined from session_id alone — defaults to `'unknown'`. Privacy level cannot be determined (the same person_id buffer contains both guild and DM turns) — defaults to `'private'` as the safe direction. Over-restrictive but correct: memories extracted here will surface in DM contexts but not in guild, preventing any accidental DM exposure.

```sql
SELECT
  h.id,
  pers.id                                                AS person_id,
  h.message                                              AS content,
  'unknown'                                              AS source_platform,
  'private'                                              AS privacy_level,
  h.created_at,
  COALESCE(pers.display_name, 'Unknown')                 AS speaker_name
FROM n8n_chat_histories h
LEFT JOIN persons pers
  -- session_id is bare person_id UUID; guard against any legacy prefixed keys
  -- still in the table from Phase 3 by only joining rows that look like UUIDs
  ON h.session_id ~ '^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$'
 AND pers.id = h.session_id::uuid
WHERE h.created_at >= NOW() - INTERVAL '2 hours'
  AND h.message IS NOT NULL AND h.message != ''
  AND h.type = 'human'   -- skip AI turns
ORDER BY h.created_at ASC
LIMIT 200;
```

**INSERT guard:** The `INSERT INTO memories` step (node 9) uses `$1::uuid` for `person_id`. If `pers.id` is NULL (session_id not a valid person UUID), skip that row — add an IF node before the INSERT: `{{ $json.person_id != null }}`. Do NOT insert memories with NULL person_id.

const group = $input.first().json;
const transcript = group.messages.map(m =>
  `[${m.speaker_name || 'Unknown'}]: ${m.content}`

const body = {
  model: 'anthropic/claude-haiku-4',
  temperature: 0.1,
  max_tokens: 800,
  messages: [
    {
      role: 'system',
      content: `You extract structured observations from conversation transcripts. Return a JSON array only — no preamble, no explanation.

Extract:
- factual_statements: stated facts about who people are (job, location, relationships, name)
- emotional_moments: frustration, excitement, distress (include emotion_type)
- decisions_made: choices someone committed to
- technical_content: code, configs, technical specs — VERBATIM, never paraphrase
- interests: topics of genuine engagement (games, movies, hobbies, projects)

Return format:
[{"observation_type":"factual_statement|emotional_moment|decision|technical_content|interest","key_label":"short.snake.key","value_text":"extracted value","asserted_by":"self|third_party","speaker_id_hint":"name if identifiable","confidence":0.0}]

If no observations worth storing, return []`
    },
    {
      role: 'user',
      content: `Extract observations from this conversation:\n\n${transcript}`
    }
  ]
};
