<task type="auto">
  <name>Task 1: Create ~/aerys/ directory structure, Docker Compose file, and environment config</name>
  <files>
    ~/aerys/docker-compose.yml
    ~/aerys/.env
    ~/aerys/.gitignore

  <action>
1. Check if Docker is installed: `docker --version && docker compose version`. If not installed, install Docker Engine + Compose plugin using the official Debian apt repo method documented in the research file (see 01-RESEARCH.md "Installation" section). After install, verify with `docker run hello-world`.

2. Create the directory structure:
```bash
mkdir -p ~/aerys/data/postgres
mkdir -p ~/aerys/config/n8n
mkdir -p ~/aerys/migrations
mkdir -p ~/aerys/workflows
chmod 755 ~/aerys/config/n8n
```

3. Generate secrets and create `~/aerys/.env`:
```
POSTGRES_USER=n8n_admin
POSTGRES_PASSWORD=<generate with: openssl rand -hex 24>
N8N_ENCRYPTION_KEY=<generate with: openssl rand -hex 32>
N8N_BASIC_AUTH_USER=aerys-admin
N8N_BASIC_AUTH_PASSWORD=<generate with: openssl rand -hex 16>
```
Use `openssl rand -hex` to generate each value dynamically. Do NOT hardcode placeholder values.

4. Create `~/aerys/docker-compose.yml` based on the official n8n-hosting template adapted for Aerys. Key specifications:
   - **postgres service:** image `pgvector/pgvector:pg16`, restart `unless-stopped`, env vars from `.env`, bind-mount `~/aerys/data/postgres:/var/lib/postgresql/data`, bind-mount `~/aerys/migrations:/docker-entrypoint-initdb.d`, healthcheck with `pg_isready`, resource limits (cpus: 2.0, memory: 2G, reservations: cpus 0.5, memory 512M)
   - **n8n service:** image `docker.n8n.io/n8nio/n8n:latest`, restart `unless-stopped`, DB_TYPE=postgresdb pointing to postgres service on port 5432 database `n8n`, N8N_ENCRYPTION_KEY from env, N8N_BASIC_AUTH_ACTIVE=true with user/password from env, EXECUTIONS_DATA_PRUNE=true with EXECUTIONS_DATA_MAX_AGE=720 and EXECUTIONS_DATA_PRUNE_MAX_COUNT=10000, port 5678:5678, bind-mount `~/aerys/config/n8n:/home/node/.n8n`, depends_on postgres with condition service_healthy, resource limits (cpus: 2.0, memory: 1G, reservations: cpus 0.25, memory 256M)
   - Do NOT include a `version:` field (deprecated in Docker Compose v2)
   - Use `${HOME}` for bind-mount paths so the compose file is portable across users

5. Create `~/aerys/.gitignore`:
```
.env
data/
config/
```
This prevents secrets and persistent data from being committed. The `migrations/`, `workflows/`, and `docker-compose.yml` files SHOULD be committed.

IMPORTANT: All environment variable references in docker-compose.yml must use `${VAR_NAME}` syntax so they're interpolated from the .env file. Do NOT hardcode any secrets.
  </action>
  <verify>
- `ls -la ~/aerys/` shows docker-compose.yml, .env, .gitignore, and subdirectories (data/, config/, migrations/, workflows/)
- `cat ~/aerys/.env` shows generated secrets (no placeholder values)
- `docker compose -f ~/aerys/docker-compose.yml config` validates the compose file without errors
  </verify>
  <done>
~/aerys/ directory exists with docker-compose.yml, .env with generated secrets, .gitignore, and all required subdirectories. Docker Compose config validation passes.
  </done>

<task type="auto">
  <name>Task 2: Create SQL migration files and start the Docker Compose stack</name>
  <files>
    ~/aerys/migrations/000_extensions.sql
    ~/aerys/migrations/001_init.sql
  </files>
  <action>
1. Create `~/aerys/migrations/000_extensions.sql`:
   - This runs first (alphabetically) when Postgres initializes
   - Connected to the default database (n8n, set by POSTGRES_DB) as superuser
   - Create the `aerys` database: `CREATE DATABASE aerys;`
   - Switch to aerys database: `\c aerys`
   - Enable extensions: `CREATE EXTENSION IF NOT EXISTS vector;` and `CREATE EXTENSION IF NOT EXISTS "uuid-ossp";`
   - Grant schema permissions for future use: `GRANT ALL ON SCHEMA public TO PUBLIC;`

2. Create `~/aerys/migrations/001_init.sql`:
   - Must begin with `\c aerys` to connect to the aerys database
   - Create four tables exactly as specified in the research file (01-RESEARCH.md "001_init.sql" section):

   **persons** table: id (UUID PK, gen_random_uuid), display_name (TEXT NOT NULL), discord_id (TEXT UNIQUE), telegram_id (TEXT UNIQUE), email (TEXT UNIQUE), timezone (TEXT), preferences (JSONB default '{}'), relationship_notes (TEXT), interaction_notes (TEXT), important_dates (JSONB default '{}'), custom_fields (JSONB default '{}'), created_at (TIMESTAMPTZ default NOW()), updated_at (TIMESTAMPTZ default NOW()), deleted_at (TIMESTAMPTZ)

   **conversations** table: id (UUID PK), person_id (UUID FK to persons ON DELETE SET NULL), channel (TEXT NOT NULL), channel_thread_id (TEXT), summary (TEXT), started_at (TIMESTAMPTZ default NOW()), last_message_at (TIMESTAMPTZ default NOW()), ended_at (TIMESTAMPTZ), deleted_at (TIMESTAMPTZ)

   **messages** table: id (UUID PK), conversation_id (UUID NOT NULL FK to conversations ON DELETE CASCADE), person_id (UUID FK to persons ON DELETE SET NULL), channel (TEXT NOT NULL), role (TEXT NOT NULL CHECK IN user/assistant/system), content (TEXT NOT NULL), content_type (TEXT default 'text'), raw_metadata (JSONB default '{}'), created_at (TIMESTAMPTZ default NOW()), deleted_at (TIMESTAMPTZ)

   **memories** table: id (UUID PK), person_id (UUID FK to persons ON DELETE SET NULL), source_message_id (UUID FK to messages ON DELETE SET NULL), content (TEXT NOT NULL), summary (TEXT), category (TEXT[]), embedding (vector(1536)), channel (TEXT), created_at (TIMESTAMPTZ default NOW()), updated_at (TIMESTAMPTZ default NOW()), deleted_at (TIMESTAMPTZ)

   - Create all indexes as specified in the research file: partial indexes on persons platform IDs, indexes on conversations (person_id, channel), indexes on messages (conversation_id, person_id, channel, created_at DESC), GIN index on memories category, partial index on memories deleted_at, and HNSW index on memories embedding with vector_cosine_ops

3. Start the Docker Compose stack:
```bash
cd ~/aerys && docker compose up -d
```

4. Wait for containers to be healthy (up to 60 seconds):
```bash
docker compose -f ~/aerys/docker-compose.yml ps
```
Check that both postgres and n8n containers show as "running" (postgres should be "healthy").

5. Verify the aerys database and schema:
```bash
# Verify aerys database exists
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "SELECT 1;"

# Verify pgvector extension
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "SELECT extname FROM pg_extension WHERE extname = 'vector';"

# Verify all four tables exist
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "\dt"

# Verify vector column and HNSW index on memories
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "\d memories"
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "SELECT indexname FROM pg_indexes WHERE tablename = 'memories' AND indexdef LIKE '%hnsw%';"
```

6. Verify n8n is running and connected to Postgres:
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:5678/
```
Should return 200 (or 401 if basic auth redirects to login).

NOTE on container naming: Docker Compose names containers as `{project}-{service}-{number}`. The project name defaults to the directory name (`aerys`), so containers will be `aerys-postgres-1` and `aerys-n8n-1`. If the naming differs, adjust commands accordingly.
  </action>
  <verify>
- `docker compose -f ~/aerys/docker-compose.yml ps` shows both containers running
- `docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "\dt"` shows persons, conversations, messages, memories tables
- `docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "SELECT extname FROM pg_extension WHERE extname = 'vector';"` returns "vector"
- `docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c "SELECT indexname FROM pg_indexes WHERE tablename = 'memories' AND indexdef LIKE '%hnsw%';"` returns the HNSW index
- `curl -s -o /dev/null -w "%{http_code}" http://localhost:5678/` returns 200 or 401
  </verify>
  <done>
Docker Compose stack is running with PostgreSQL 16 + pgvector and n8n. Containers restart automatically via unless-stopped policy.
  </done>
