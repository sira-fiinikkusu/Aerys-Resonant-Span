1. **Test data persistence through restart cycle:**
```bash
# Insert a test row
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c \
  "INSERT INTO persons (display_name) VALUES ('__persistence_test__');"

# Full stop and start (not just restart — down destroys containers)
cd ~/aerys && docker compose down && docker compose up -d

# Wait for healthy
sleep 10

# Verify test row survived
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c \
  "SELECT display_name FROM persons WHERE display_name = '__persistence_test__';"

# Clean up test row
docker exec aerys-postgres-1 psql -U n8n_admin -d aerys -c \
  "DELETE FROM persons WHERE display_name = '__persistence_test__';"
```
If the test row is NOT found after restart, the bind-mount is misconfigured — debug the volume mapping in docker-compose.yml.

2. **Discover NAS mount point:**
```bash
# Check common NAS mount locations
mount | grep -i nas
ls /mnt/nas 2>/dev/null || ls /media/nas 2>/dev/null || df -h | grep -i nas
```
If NAS is not mounted, check if there's a known mount path or network share. Create the backup directory at whatever path is discovered. If NAS is not available, use a local backup path at `~/aerys/backups/` as fallback and note it in the summary for the user to configure NAS later.

3. **Create backup script** at `~/aerys/backup.sh`:
```bash
#!/bin/bash
# Aerys daily database backup
# Backs up both aerys and n8n databases from the shared Postgres container

BACKUP_DIR="${BACKUP_DIR:-/mnt/nas/aerys-backups}"  # Override via env if NAS path differs
RETENTION_DAYS=7
DATE=$(date +%Y%m%d_%H%M%S)
CONTAINER="aerys-postgres-1"
PG_USER="n8n_admin"

mkdir -p "$BACKUP_DIR"

# Backup aerys database
docker exec "$CONTAINER" pg_dump -U "$PG_USER" aerys | gzip > "$BACKUP_DIR/aerys_${DATE}.sql.gz"

# Backup n8n database
docker exec "$CONTAINER" pg_dump -U "$PG_USER" n8n | gzip > "$BACKUP_DIR/n8n_${DATE}.sql.gz"

# Prune old backups
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +${RETENTION_DAYS} -delete

echo "[$(date)] Backup complete: aerys_${DATE}.sql.gz, n8n_${DATE}.sql.gz"
```
Make it executable: `chmod +x ~/aerys/backup.sh`

Adjust `BACKUP_DIR` default based on the actual NAS mount point discovered in step 2. If NAS is not available, default to `~/aerys/backups/`.

4. **Run the backup script once** to verify it works:
```bash
~/aerys/backup.sh
ls -la ${BACKUP_DIR}/*.sql.gz
```

5. **Add cron entry for daily 3 AM backup:**
```bash
# Add to crontab (append, don't overwrite existing entries)
(crontab -l 2>/dev/null; echo "0 3 * * * ${HOME}/aerys/backup.sh >> ${HOME}/aerys/backup.log 2>&1") | crontab -
```
Verify: `crontab -l` should show the backup entry.
