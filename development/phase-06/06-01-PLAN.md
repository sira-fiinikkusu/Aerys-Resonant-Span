<task type="auto">
  <name>Task 2: Build LLM-as-judge eval workflow in n8n</name>
  <files>n8n eval workflow (new), ~/aerys/workflows/06-01-eval-suite.json</files>
  <action>
Create a new n8n workflow "06-01 Eval Suite" via n8n API. The workflow implements the LLM-as-judge pattern:

**Workflow structure:**
```
Manual Trigger → Load Dataset (Code: read baseline.json via require('fs'))
  → SplitInBatches (batch=1)
    → Build Test Input (Code: format user_message + context for Core Agent)
    → Execute Core Agent (Execute Workflow, workflowId YOUR_CORE_AGENT_WORKFLOW_ID)
    → LLM Judge (HTTP Request to OpenRouter, Sonnet model, judge prompt)
    → Parse Score (Code: extract score 1-5 + reasoning)
    → Collect Results (Code: accumulate scores in staticData)
    → loop back to SplitInBatches
  → [done branch] Format Report (Code: average score, per-category breakdown, min/max)
```

**CRITICAL n8n quirks to handle:**
- SplitInBatches v3: branch 0 = done, branch 1 = loop. Connect processing to branch 1.
- Execute Workflow typeVersion 2: `{workflowId: {value: "YOUR_CORE_AGENT_WORKFLOW_ID", mode: "id"}}` format, NOT __rl.
- Use Code nodes for all data manipulation (Set node returns {} in this n8n version).
- LLM Judge HTTP Request: use OpenRouter credential YOUR_OPENROUTER_CREDENTIAL_ID with model `anthropic/claude-sonnet-4-5`.
- Execute Workflow must pass workflowInputs with mappingMode "defineBelow" including at minimum: source_channel, person_id, session_id, conversation_privacy. The Core Agent expects these from adapters.

**LLM Judge system prompt:**
```
You are an expert evaluator for an AI assistant named Aerys.
Given a user message and Aerys's response, score the response 1-5:
5 = Perfect: accurate, natural, in Aerys's voice, directly addresses the question
4 = Good: mostly correct, minor issues with tone or completeness
3 = Acceptable: correct but mechanical, missing personality, or slightly off-topic
2 = Poor: partially wrong, ignores the question, or very out of character
1 = Fail: wrong, harmful, or complete non-answer

Respond with JSON only:
{"extended_reasoning": "<analysis>", "reasoning_summary": "<1 sentence>", "score": <1-5>}
```

**Build Test Input Code node** must construct the minimum payload the Core Agent expects:
```javascript
const testCase = $json;
return [{json: {
  content: testCase.user_message,
  source_channel: 'discord',
  person_id: testCase.context?.person_id || '00000000-0000-0000-0000-000000000001',
  session_id: testCase.context?.person_id || '00000000-0000-0000-0000-000000000001',
  conversation_privacy: 'private',
  username: 'eval_runner',
  display_name: 'Eval Runner',
  _start_ts: Date.now()
}}];
```

**Format Report Code node** outputs a summary:
```javascript
const staticData = $getWorkflowStaticData('global');
const scores = staticData.scores || [];
const avg = scores.reduce((a,b) => a+b, 0) / scores.length;
// Per-category breakdown
const byCategory = {};
(staticData.details || []).forEach(d => {
  if (!byCategory[d.category]) byCategory[d.category] = [];
  byCategory[d.category].push(d.score);
});
// Clear for next run
staticData.scores = [];
staticData.details = [];
return [{json: {average_score: avg.toFixed(2), total_cases: scores.length, by_category: byCategory}}];
```
