## Accomplishments
- Extraction prompt rewritten with explicit REJECT list filtering transient actions, session decisions, and meta-commentary

## Decisions Made
- Extraction prompt REJECT list covers: transient actions (going to bed), session decisions (parking bugs), meta-commentary (system discussions), greetings, acknowledgments
- event_date defaults to batch message created_at when LLM returns null -- decided during verification when initial extraction showed null dates for all memories
- key_label backfilled from existing content column using SPLIT_PART on colon separator -- enables plan 02 dedup without full re-extraction
- Content format string: `key_label: value_text -- context (event_date)` chosen as single-string format readable by both humans and retrieval pipeline

- **Files modified:** ~/aerys/workflows/04-02-memory-batch.json (Parse Observations Code node)
