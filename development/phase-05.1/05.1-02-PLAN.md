Add write-time deduplication to the extraction pipeline and update the retrieval display format so memories read naturally without key_label prefixes.

Purpose: Without dedup, the same fact extracted in consecutive hourly runs creates duplicate rows. The retrieval display update removes the machine-readable key_label prefix so the Core Agent sees natural language.

**Part C: Add Route Dedup Result Code node**

Add a new Code node named "Route Dedup Result" after Dedup Check. This node determines the action (insert, update, or replace) based on dedup results.

Logic (from RESEARCH.md "Dedup Check + Route Logic"):
```javascript
const dupResult = $input.first().json;
const obs = $('Parse Observations').item.json;

// Sentinel row check (UNION ALL pattern)
const hasDuplicate = dupResult.id != null;

if (hasDuplicate) {
  // Compare value to decide: update-in-place vs replace
  const existingValue = (dupResult.content || '').split(':').slice(1).join(':').trim();
  // Strip context suffix for comparison (content format: "key: value -- context (date)")
  const existingCore = existingValue.split('--')[0].trim().toLowerCase();
  const newValue = (obs.value_text || '').toLowerCase();

  const valueChanged = existingCore !== newValue
    && !existingCore.includes(newValue)
    && !newValue.includes(existingCore);

  return [{
    json: {
      action: valueChanged ? 'replace' : 'update',
      existing_id: dupResult.id,
      person_id: obs.person_id,
      key_label: obs.key_label,
      value_text: obs.value_text,
      context: obs.context || null,
      event_date: obs.event_date || null,
      privacy_level: obs.privacy_level || 'public',
      source_platform: obs.source_platform || 'discord',
      new_content: obs.new_content
    }
  }];
} else {
  return [{
    json: {
      action: 'insert',
      person_id: obs.person_id,
      key_label: obs.key_label,
      value_text: obs.value_text,
      context: obs.context || null,
      event_date: obs.event_date || null,
      privacy_level: obs.privacy_level || 'public',
      source_platform: obs.source_platform || 'discord',
      new_content: obs.new_content
    }
  }];
}
```

**Part H: Update Format Memory Context in retrieval workflow**

In workflow YOUR_MEMORY_RETRIEVAL_WORKFLOW_ID, update the Format Memory Context Code node to strip the key_label prefix from content before display. Use the code from RESEARCH.md "Updated Format Memory Context" section:

```javascript
const input = $input.first().json;
const rows = input.rows || [];

if (rows.length === 0) {
  return [{ json: { memory_context: '' } }];
}

const seen = new Set();
const deduped = rows.filter(r => {
  const key = r.content && r.content.includes(':')
    ? r.content.split(':')[0].trim().toLowerCase()
    : r.content ? r.content.trim().toLowerCase() : '';
  if (seen.has(key)) return false;
  seen.add(key);
  return true;
});

const top = deduped.slice(0, 5);

const lines = top.map((r) => {
  let display = r.content || '';
  if (display.includes(':')) {
    display = display.split(':').slice(1).join(':').trim();
  }
  const age = r.created_at
    ? `(${Math.round((Date.now() - new Date(r.created_at)) / 86400000)}d ago)`
    : '';
  const src = r.source_platform ? `[${r.source_platform}]` : '';
  return `* ${display} ${src} ${age}`.trim();
});

return [{ json: { memory_context: lines.join('\n') } }];
```
