This is a dedicated workflow called by the core agent via Execute Workflow node — keeping the output pipeline decoupled from the reasoning pipeline for maintainability and testability.

    **Node sequence (in the 02-04-output-router workflow):**

    **0. Execute Workflow Trigger** (built-in)
    - Entry point — called by the core agent's Prepare Response node via Execute Workflow.
    - Receives the response object as workflow input data.

    **1. Polisher Gate** (Code node)
    ```javascript
    const response = $input.item.json.raw_response;
    const source = $input.item.json.source || 'core_agent';
    const wordCount = response.split(/\s+/).length;

    const needsPolish = (
      wordCount > 150 ||
      source === 'sub_agent' ||
      /i cannot|i'm sorry|unfortunately|i apologize/i.test(response)
    );

    return [{ json: { ...$input.item.json, needs_polish: needsPolish } }];
    ```

    **2. IF: Needs Polish** (IF node)
    - Condition: `{{ $json.needs_polish }}` is true
    - True branch → Polisher AI Agent
    - False branch → skip to formatter (pass raw_response as polished_response)

    **3a. Polisher AI Agent** (AI Agent node — true branch)
    - System Prompt:
      ```
      You are a tone editor for Aerys, a composed and protective AI assistant. Your ONLY job is adjusting voice and tone. Rules:
      1. Preserve ALL factual content exactly — do not add, remove, or change information
      2. Do NOT modify code blocks or technical content
      3. Adjust voice to match Aerys's character: composed, direct, curious, protective
      4. Remove any apologetic or overly deferential language ("I'm sorry", "Unfortunately", "I apologize")
      5. Replace with Aerys's natural patterns: direct acknowledgment → immediate path forward
      6. Keep response length approximately the same
      7. Return ONLY the revised text, no commentary
      ```
    - User Message: `{{ $json.raw_response }}`
    - Sub-node: OpenRouter Chat Model with Haiku (polishing is a light task, Haiku is sufficient)
    - No memory sub-node needed (polisher is stateless)

    **3b. Skip Polisher** (Set node — false branch)
    - Set `polished_response` = `{{ $json.raw_response }}`

    **4. Merge** (Merge node — rejoin polished and unpolished branches)

    **5. Platform Formatter** (Code node)
    ```javascript
    const text = $input.item.json.polished_response || $input.item.json.output || $input.item.json.raw_response;
    const platform = $input.item.json.source_channel;

    function toTelegramHtml(md) {
      return md
        // Code blocks first (before inline code)
        .replace(/```(\w+)?\n([\s\S]*?)```/g, '<pre><code>$2</code></pre>')
        // Inline code
        .replace(/`([^`]+)`/g, '<code>$1</code>')
        // Bold
        .replace(/\*\*([^*]+)\*\*/g, '<b>$1</b>')
        // Italic (after bold to avoid conflict)
        .replace(/(?<!\*)\*([^*]+)\*(?!\*)/g, '<i>$1</i>');
    }

    let formatted;
    let parseMode;

    if (platform === 'telegram') {
      formatted = toTelegramHtml(text);
      parseMode = 'HTML';
    } else {
      // Discord: markdown passes through natively, no conversion needed
      formatted = text;
      parseMode = undefined;
    }

    return [{ json: { ...$input.item.json, formatted_response: formatted, parse_mode: parseMode } }];
    ```

    **6. Message Splitter** (Code node)
    ```javascript
    const text = $input.item.json.formatted_response;
    const platform = $input.item.json.source_channel;
    const limit = platform === 'discord' ? 2000 : 4096;

    function splitMessage(text, limit) {
      if (text.length <= limit) return [text];
      const chunks = [];
      let remaining = text;
      while (remaining.length > limit) {
        let splitAt = remaining.lastIndexOf('\n\n', limit);
        if (splitAt < limit * 0.5) splitAt = remaining.lastIndexOf('\n', limit);
        if (splitAt < limit * 0.5) splitAt = remaining.lastIndexOf('. ', limit);
        if (splitAt < 0 || splitAt < limit * 0.3) splitAt = limit;
        chunks.push(remaining.slice(0, splitAt).trim());
        remaining = remaining.slice(splitAt).trim();
      }
      if (remaining) chunks.push(remaining);
      return chunks;
    }

    const chunks = splitMessage(text, limit);
    return chunks.map((chunk, i) => ({
      json: {
        ...$input.item.json,
        chunk,
        chunk_index: i,
        total_chunks: chunks.length
      }
    }));
    ```

    **7. Switch: Route by Platform** (Switch node)
    - Route on `{{ $json.source_channel }}`
    - Outputs: "discord", "telegram"

    **8a. Send Discord Message** (Discord Send Message node from community package, or HTTP Request node)
    - Using HTTP Request node for more control:
      - Method: POST
      - URL: `https://discord.com/api/v10/channels/{{ $json.channel_id }}/messages`
      - Headers: Authorization: `Bot {{ DISCORD_BOT_TOKEN }}`
      - Body: `{ "content": "{{ $json.chunk }}" }`
    - For Discord embeds (structured output): detect if classification.task_type is "research" or "analysis" AND response is structured — use embed format instead of plain content. Implementation:
      ```javascript
      // If task is research/analysis and response has sections, use embed
      const isStructured = ['research', 'analysis'].includes($json.classification?.task_type);
      if (isStructured && $json.chunk_index === 0) {
        // Use embed format for first chunk
        body = {
          embeds: [{
            title: "Research Results",
            description: $json.chunk,
            color: 0x5865F2  // Discord blurple
          }]
        };
      } else {
        body = { content: $json.chunk };
      }
      ```

    **8b. Send Telegram Message** (built-in Telegram node — Send Message operation)
    - Chat ID: `{{ $json.channel_id }}`
    - Text: `{{ $json.chunk }}`
    - Parse Mode: `{{ $json.parse_mode }}` (HTML)
    - Disable link preview: true (to avoid unfurling URLs in AI responses)

    **9. Loop for multi-chunk messages:**
    The Message Splitter (step 6) outputs multiple items if the response exceeds the platform limit. n8n automatically loops over each item through the downstream nodes, so each chunk is sent as a separate message. No explicit loop node needed — n8n's item-based execution handles this.

    **Wiring into core agent:**
    - Add an Execute Workflow node after the Prepare Response node in the core agent workflow (02-03) that calls 02-04-output-router
    - Pass the full response object as workflow input data
    - The output router sends messages directly to Discord/Telegram — it does NOT return to the adapter. The adapter's Execute Workflow call simply waits for completion.

    Export workflow JSON to `~/aerys/workflows/02-04-output-router.json` and update `~/aerys/workflows/02-03-core-agent.json` with the new Execute Workflow node.
