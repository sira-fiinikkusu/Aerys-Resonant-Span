<task type="auto">
  <name>Task 1: Build core agent workflow — intent classifier, model router, and AI Agent</name>
  <files>
    ~/aerys/workflows/02-03-core-agent.json
  </files>
  <action>
    Build a single n8n workflow named "02-03-core-agent" that is called by the channel adapter workflows via Execute Workflow node. This workflow receives a normalized message object and returns an AI response.

    **Workflow input:** Normalized message object from channel adapters (fields: source_channel, channel_id, guild_id, user_id, username, message_text, message_id, is_dm, is_mention, attachments, timestamp, session_key).

    **Node sequence:**

    **1. Execute Workflow Trigger** (built-in)
    - This is the entry point — called by Discord/Telegram adapter workflows.
    - Receives the normalized message as workflow input data.

    **2. Read Soul Prompt** (Read/Write Files from Disk node)
    - Operation: Read File(s) From Disk
    - File Path: `/home/node/aerys-config/soul.md`
    - Output: As String
    - This runs once per workflow execution. The soul prompt string is passed to the AI Agent system prompt.

    **3. Read Model Config** (Read/Write Files from Disk node)
    - File Path: `/home/node/aerys-config/models.json`
    - Output: As String
    - Parse in next node.

    **4. Intent Classifier** (AI Agent node — simple, non-tools)
    - Use a basic HTTP Request node to OpenRouter (not a full AI Agent node) for the classifier. This avoids overhead.
    - Alternative: Use a dedicated AI Agent node with Haiku.

    Better approach — use a **Code node** that calls the OpenRouter API directly via HTTP:
    ```javascript
    // Intent classifier via OpenRouter Haiku
    const apiKey = $credentials.openRouterApi.apiKey; // or from env
    const message = $input.item.json.message_text;

    const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        model: 'anthropic/claude-haiku-4.5',
        messages: [
          {
            role: 'system',
            content: 'Classify the user message. Output ONLY valid JSON, no other text:\n{"task_type": "<greeting|simple_qa|code_help|research|creative|analysis|system_task>", "confidence": <0.0-1.0>, "suggested_model": "<haiku|sonnet|opus>"}'
          },
          { role: 'user', content: message }
        ],
        max_tokens: 100,
        temperature: 0
      })
    });
    const data = await response.json();
    const classification = JSON.parse(data.choices[0].message.content);
    return [{ json: { ...$input.item.json, classification } }];
    ```

    ACTUALLY — prefer using n8n's built-in HTTP Request node instead of Code node fetch, for better error visibility in n8n UI. Use an **HTTP Request node**:
    - Method: POST
    - URL: `https://openrouter.ai/api/v1/chat/completions`
    - Authentication: use Header Auth with `Authorization: Bearer {{OPENROUTER_API_KEY}}`
    - Body (JSON): the classifier prompt with `model: "anthropic/claude-haiku-4.5"`
    - Response: JSON parsed

    **5. Parse Classification + Cost Guard** (Code node)
    ```javascript
    // Parse classifier output
    const raw = $input.item.json;
    const classifierResponse = raw.data?.choices?.[0]?.message?.content || '{"task_type":"simple_qa","confidence":0.5,"suggested_model":"sonnet"}';
    let classification;
    try {
      classification = JSON.parse(classifierResponse);
    } catch (e) {
      classification = { task_type: "simple_qa", confidence: 0.5, suggested_model: "sonnet" };
    }

    // Pass classification through — cost guard applied in next node after DB check
    return [{ json: { ...raw, classification } }];
    ```

    **6. Check Opus Daily Usage** (Postgres node)
    - Query: `SELECT COALESCE(call_count, 0) AS count FROM aerys_model_usage WHERE date = CURRENT_DATE AND model = 'opus'`
    - Credential: aerys database credential (host: postgres, port: 5432, db: aerys, user: postgres)
    - If no rows: count = 0

    **7. Resolve Model + Apply Cost Guard** (Code node)
    ```javascript
    const input = $input.item.json;
    const classification = input.classification;
    const opusCount = $('Check Opus Daily Usage').item.json.count || 0;
    const modelsConfig = JSON.parse($('Read Model Config').item.json.data);
    const opusLimit = modelsConfig.limits?.opus_daily || 10;

    let modelTier = classification.suggested_model || 'sonnet';

    // Cost guard: degrade opus → sonnet when cap hit
    if (modelTier === 'opus' && opusCount >= opusLimit) {
      modelTier = 'sonnet';
    }

    const modelId = modelsConfig.models[modelTier] || modelsConfig.models.sonnet;

    return [{
      json: {
        ...input,
        resolved_model_id: modelId,
        model_tier: modelTier,
        opus_count: opusCount,
        opus_limit: opusLimit
      }
    }];
    ```

    **8. Switch: Route by Model Tier** (Switch node)
    - Route on `{{ $json.model_tier }}`
    - Outputs: "sonnet" (default), "opus", "haiku"
    - Each output connects to a dedicated AI Agent node with the corresponding OpenRouter Chat Model sub-node configured for that model ID.

    **9a/9b/9c. AI Agent nodes** (one per model tier — sonnet, opus, haiku)
    Each AI Agent node:
    - Type: Tools Agent (the only type in n8n 1.82+)
    - System Prompt: Content of soul.md (injected from the Read Soul Prompt node output). Append the speaker-tagging instruction:
      ```
      {{soulPrompt}}

      Previous conversation is speaker-tagged:
      [username]: user messages
      [Aerys]: your responses

      The user's name is: {{$json.username}}
      ```
    - User Message: `{{ $json.message_text }}`
    - Sub-nodes:
      - **OpenRouter Chat Model** sub-node: model ID set to the resolved model for this branch (e.g., `anthropic/claude-sonnet-4.5` for the sonnet branch)
      - **Postgres Chat Memory** sub-node:
        - Credential: aerys database (NOT the n8n database)
        - Session Key: `{{ $json.session_key }}` (e.g., `discord_1234567890`)
        - Context Window Length: 60

    **CRITICAL:** Create a separate Postgres credential for the aerys DB in n8n. Do NOT reuse the n8n DB credential. The Postgres Chat Memory node creates `n8n_chat_histories` table in whichever DB the credential points to.

    **10. Merge Model Outputs** (Merge node or simply wire all three AI Agent outputs to the same next node)
    - All three AI Agent branches converge here.
    - Pass through the AI Agent output plus the original normalized message fields.

    **11. Increment Opus Counter** (Postgres node — conditional)
    - Only execute if `model_tier === 'opus'`
    - Use an IF node before this: condition `{{ $json.model_tier }}` equals "opus"
    - Query: `INSERT INTO aerys_model_usage (date, model, call_count) VALUES (CURRENT_DATE, 'opus', 1) ON CONFLICT (date, model) DO UPDATE SET call_count = aerys_model_usage.call_count + 1`

    **12. Prepare Response** (Code node)
    ```javascript
    const input = $input.item.json;
    return [{
      json: {
        // Original normalized fields for output routing
        source_channel: input.source_channel,
        channel_id: input.channel_id,
        guild_id: input.guild_id,
        user_id: input.user_id,
        username: input.username,
        message_id: input.message_id,
        is_dm: input.is_dm,
        session_key: input.session_key,
        // AI response
        raw_response: input.output || input.text || "",
        model_tier: input.model_tier,
        resolved_model_id: input.resolved_model_id,
        classification: input.classification,
        // Source flag for polisher gate
        source: "core_agent"
      }
    }];
    ```

    **13. Return Response** (the workflow output — flows back to the calling Execute Workflow node in the adapter)

    Export workflow JSON to `~/aerys/workflows/02-03-core-agent.json`.

    **Fallback chain implementation:**
    The Switch node routes to the selected model. If a model call fails (OpenRouter returns error), the AI Agent node will error. Handle this with n8n's built-in error handling:
    - On each AI Agent node, set "Continue On Fail" = true
    - After each AI Agent, add an IF node: if output is empty or has error, route to the next tier down (Opus → Sonnet AI Agent, Sonnet → Haiku AI Agent)
    - Final fallback: if Haiku also fails, return a static error message: "I'm having trouble connecting to my reasoning systems right now. Give me a moment and try again."

    **Wire the channel adapters:**
    After creating this workflow, update the Execute Workflow nodes in the Discord adapter (02-01) and Telegram adapter (02-02) to point to this workflow by name or ID.
