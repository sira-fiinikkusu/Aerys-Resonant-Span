    Build two n8n workflows via the n8n REST API (or import JSON). Each workflow is a channel adapter that triggers on incoming messages and normalizes them to a standard internal format.

    **Discord Adapter Workflow (02-01-discord-adapter.json):**

    Nodes:
    1. **Discord Trigger** (`@kmcbride3/n8n-nodes-discord` trigger node)
       - Event: Message Created
       - Bot token: from n8n Discord App credential (user creates credential with DISCORD_BOT_TOKEN)

    2. **Filter: Should Respond** (Code node)
       - In DMs: always respond (is_dm = true)
       - In servers: only respond if bot is @mentioned (check raw.content for `<@BOT_ID>`)
       - Ignore messages from bots (raw.author.bot === true)
       - **Edit detection:** Check `raw.editedTimestamp` — if present, the message is an edit.
         - If edit + bot @mentioned: route through with `is_edit: true` flag on the normalized output (acknowledge path)
         - If edit without @mention: drop the message (filter out)
       - Pass through: messages that should get a response (new messages + edit-with-mention)
       - Filter out: messages in servers without @mention, bot messages, edits without @mention

    3. **Normalize Message** (Code node)
       ```javascript
       const raw = $input.item.json;
       const botId = ""; // Set via credential or env — filled at runtime
       const normalized = {
         source_channel: "discord",
         channel_id: raw.channelId,
         guild_id: raw.guildId || null,
         user_id: raw.author?.id,
         username: raw.author?.username || raw.author?.globalName || "unknown",
         message_text: raw.content.replace(/<@\d+>/g, '').trim(),
         message_id: raw.id,
         is_dm: !raw.guildId,
         is_mention: true,
         is_edit: !!raw.editedTimestamp,
         attachments: (raw.attachments || []).map(a => ({
           type: a.contentType || "unknown",
           size: a.size,
           url: a.url,
           filename: a.name
         })),
         timestamp: raw.createdTimestamp || Date.now(),
         session_key: `discord_${raw.channelId}`
       };
       return [{ json: normalized }];
       ```
       Note: Strip `<@BOT_ID>` from message_text so AI doesn't see raw mention syntax.

    4. **Send Typing Indicator** (HTTP Request node)
       - Method: POST
       - URL: `https://discord.com/api/v10/channels/{{$json.channel_id}}/typing`
       - Headers: Authorization: `Bot {{DISCORD_BOT_TOKEN}}`
       - This fires once; acceptable for Phase 2 (indicator expires after 10s — upgrade to loop pattern in Phase 6)

    5. **Execute Core Agent Workflow** (Execute Workflow node)

       - Pass: normalized message object
       - Mode: Wait for sub-workflow to finish

    6. **Placeholder: Output handled by core agent return** (Set node — no-op for now, output routing built in Plan 02-03)

    **Telegram Adapter Workflow (02-02-telegram-adapter.json):**

    Nodes:
    1. **Telegram Trigger** (built-in `n8n-nodes-base.telegramtrigger`)
       - Events: Message
       - Bot token: from n8n Telegram API credential (user creates credential with TELEGRAM_BOT_TOKEN)

    2. **Filter: Should Respond** (Code node)
       - Private chats (chat.type === "private"): always respond
       - Groups: only respond if bot is @mentioned in entities (check for mention entity matching bot username)
       - Ignore empty messages (no text and no caption)
       - **Edit detection:** Check `message.edit_date` — if present, the message is an edit.
         - If edit + bot @mentioned (or edit in private chat): route through with `is_edit: true` flag on the normalized output (acknowledge path)
         - If edit in group without @mention: drop the message (filter out)

    3. **Normalize Message** (Code node)
       ```javascript
       const raw = $input.item.json;
       const msg = raw.message || raw;
       const normalized = {
         source_channel: "telegram",
         channel_id: String(msg.chat.id),
         guild_id: null,
         user_id: String(msg.from.id),
         username: msg.from.username || msg.from.first_name || "unknown",
         message_text: (msg.text || msg.caption || "").trim(),
         message_id: String(msg.message_id),
         is_dm: msg.chat.type === "private",
         is_mention: msg.chat.type === "private",
         is_edit: !!msg.edit_date,
         attachments: [],
         timestamp: (msg.date || Math.floor(Date.now()/1000)) * 1000,
         session_key: `telegram_${msg.chat.id}`
       };
       // Preserve attachment metadata for Phase 5
       if (msg.document) {
         normalized.attachments.push({
           type: msg.document.mime_type || "unknown",
           size: msg.document.file_size,
           url: null,
           filename: msg.document.file_name
         });
       }
       if (msg.photo) {
         const largest = msg.photo[msg.photo.length - 1];
         normalized.attachments.push({
           type: "image/jpeg",
           size: largest.file_size,
           url: null,
           filename: "photo.jpg"
         });
       }
       return [{ json: normalized }];
       ```

    4. **Send Typing Indicator** (Telegram node — Send Chat Action)
       - Action: typing
       - Chat ID: `{{$json.channel_id}}`

    5. **Execute Core Agent Workflow** (Execute Workflow node)
       - Workflow: "02-03-core-agent" (created in Plan 02-02)
       - Pass: normalized message object
       - Mode: Wait for sub-workflow to finish

    6. **Placeholder: Output handled by core agent return** (Set node — no-op for now)
